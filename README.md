# 台北捷運資訊系統 UI/UX 設計指引

這個專案是一個可直接用瀏覽器開啟的靜態網頁工具，整理台北捷運 CI（企業識別）相關的色彩、字體、Logo 使用與簡報版型規範，協助同仁快速查找品牌資源、建立符合規範的視覺原型，並降低跨單位溝通成本。

本工具也可視為「北捷品牌數位管家」的展示版本，適合用於提案展示、內部教育訓練、設計規範查詢，以及非技術人員快速產出符合品牌規範的系統介面雛形。

## 適合誰使用

- 第一次接觸台北捷運 CI 規範的新手使用者
- 需要查詢品牌色碼、Logo 使用方式或字體規範的同仁
- 需要製作簡報、報表、內部系統畫面或活動頁面的業務單位
- 需要與資訊或設計單位溝通 UI/UX 需求的提案人
- 想快速展示 Vibe Coding / AI 輔助開發成果的人員

## 主要功能

- **品牌色卡查詢**：整理官方品牌色，可搜尋、分類、排序並一鍵複製色碼。
- **HSV 色彩分析**：提供色相、明度、彩度資訊，方便判斷色彩搭配。
- **Logo 使用指引**：說明彩色、反白、黑字等 Logo 版本的使用情境。
- **字體規範**：整理翔鶴黑體、思源黑體等字體使用建議。
- **簡報模板參考**：提供簡報封面、內容頁與版型應用參考。
- **品牌產生器**：可即時預覽主色、輔助色、Logo、字體與模板組合。
- **版型產生器**：可依不同業務情境生成符合 CI 的系統介面原型。
- **應用範例**：提供已套用品牌視覺的互動展示頁面。

## 快速上手

本專案不需要安裝 Node.js、資料庫或後端服務，只要有瀏覽器即可使用。

### 方式一：直接開啟

1. 下載或取得本專案資料夾。
2. 在專案根目錄找到 `index.html`。
3. 用 Chrome、Edge 或其他現代瀏覽器開啟 `index.html`。
4. 看到「台北捷運資訊系統 UI/UX 設計指引」頁面後即可開始使用。

### 方式二：從 Git 取得專案

```bash
git clone <repo-url>
cd TRTC_CFS
```

接著直接用瀏覽器開啟 `index.html`。

> 若你不熟悉 Git，也可以直接下載 ZIP 檔，解壓縮後開啟 `index.html`。

## 新手使用教學

### 1. 查詢品牌色碼

進入首頁後，找到「品牌色卡」區塊。你可以：

- 點擊色卡複製 HEX 色碼。
- 使用搜尋功能查找特定顏色。
- 依色系、色相、明度或彩度查看顏色。

複製後可貼到簡報、設計工具、HTML/CSS 或其他文件中使用。

### 2. 查看字體規範

點選頁面上方或導覽列中的「字體規範」，可查看不同情境下建議使用的字體。若是網頁或內部工具，可優先參考頁面中建議的網頁字體設定。

### 3. 查看 Logo 使用方式

點選「Logo 使用」可查看 Logo 在不同背景下的建議用法。使用時請避免任意變形、裁切、旋轉、壓縮或更動字距。

### 4. 下載 Logo 素材

首頁快速導覽中有「下載 Logo」連結，可下載 `files/TRTC_logo.7z`。下載後請依公司 CI 規範選用正確版本。

### 5. 使用品牌產生器

在「品牌產生器」區塊中，可以選擇主色、輔助色、Logo 版本、字體與模板，系統會即時產生視覺預覽。這適合在正式製作前先確認整體風格。

### 6. 使用版型產生器

在「版型產生器」區塊中，可以選擇業務場景與版面結構，快速產生符合品牌風格的系統介面原型。產出的 HTML 可作為後續開發或需求討論的起點。

### 7. 查看應用範例

可開啟 `act_now/index.html` 查看實際應用範例，了解品牌視覺如何被套用到完整頁面中。

## 常見問題

### 需要安裝任何軟體嗎？

不需要。只要有瀏覽器即可開啟 `index.html` 使用。

### 可以離線使用嗎？

大部分功能可以離線使用。若瀏覽器需要載入外部字體，離線時可能會改用系統預設字體，但主要內容與互動功能仍可使用。

### 修改後要如何分享？

如果只是給同仁查看，可直接分享整個專案資料夾或壓縮成 ZIP。若要納入正式版本管理，請將修改後的檔案提交到 Git repo。

### 非工程背景的人可以用嗎？

可以。本專案設計目標之一就是讓非技術使用者能快速查詢規範、複製色碼、預覽品牌組合，並產出可討論的系統畫面雛形。

## 建議使用環境

- Chrome、Microsoft Edge 或其他現代瀏覽器
- 螢幕寬度建議 1280px 以上，可取得較完整的桌面版體驗
- 手機或平板也可瀏覽，但部分產生器操作在桌面環境較方便

## 維護提醒

- 新增品牌素材時，建議放入 `assets/` 或 `files/`，並在 README 補充用途。
- 新增提案或操作文件時，建議放入 `docs/`。
- 若調整首頁功能，請同步確認 `index.html`、`style.css`、`app.js` 三者一致。
- 若要對外正式發布，請先確認 Logo、CI 規範與文件版本皆為最新可用版本。
