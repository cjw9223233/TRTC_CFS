🤖什麼是 AI Agent？

AI Agent（AI 代理人）是一種可以自動執行多步驟任務的 AI 程式。它不只是回答問題，還能主動讀取檔案、執行指令、呼叫工具，完成你交辦的完整工作。

🔑 核心概念
一般 AI 聊天：你問一句 → AI 答一句

AI Agent：你說「幫我把這份報告整理成投影片」→ Agent 自動讀取報告、分析結構、生成投影片、回報結果，全程自動完成。

OpenAI 的 Codex 是目前最強大的 AI 編程代理，它能在 CLI（命令列）、IDE 編輯器外掛、或 Web App 中幫你自動寫程式、修 bug、跑測試。

1
你給任務（Prompt）

用自然語言描述你想做什麼

2
Agent 規劃步驟

Agent 分析任務、找出需要的工具和資訊

3
自動執行

讀檔、呼叫 API、執行指令、修改程式碼

4
回報結果

完成任務，讓你審查確認

下列哪一個描述最符合「AI Agent」與「一般 AI 聊天」的差異？

✍️寫出好的提示詞（Prompt）

提示詞不需要完美，但越清楚，Agent 的結果越可靠。OpenAI 建議一個好的 Prompt 包含四個要素：

📋 Prompt 四要素
  • 🎯 Goal（目標）：你想改什麼或做什麼？
  • 📁 Context（脈絡）：哪些檔案、錯誤訊息、範例是相關的？
  • ⚙️ Constraints（限制）：有哪些規範、架構、安全要求？
  • ✅ Done When（完成條件）：什麼狀態叫做「完成」？

你也可以根據任務難度調整 Agent 的「推理等級」：

  • Low — 快速、範圍清楚的小任務
  • Medium / High — 複雜的修改或 debug
  • Extra High — 長時間、需要深度推理的任務
💡 在 Codex App 裡，你可以用語音輸入來口述任務，不用打字！這對描述複雜需求特別方便。

📝Prompt 範例對比

❌ 不好的 Prompt：

幫我修好那個 bug

✅ 好的 Prompt：

目標：修正用戶登入失敗的 bug
脈絡：錯誤出現在 auth/login.py 第 42 行，錯誤訊息為 "KeyError: token"
限制：不可修改資料庫 schema，需維持 Python 3.9 相容性
完成條件：所有登入相關的單元測試通過

🗺️先規劃，再行動（Plan First）

面對複雜或模糊的任務，不要急著讓 Agent 直接開始做，而是先讓它「規劃」再「執行」。這樣可以大幅降低走錯方向的風險。

🔑 為什麼要先規劃？
想像你要蓋一棟房子，你不會直接叫工人開始動工，而是先看設計圖、確認需求、再拿到施工許可。AI Agent 也一樣 — 複雜任務需要先形成計畫。

OpenAI 提供三種規劃方式：

1
Plan Mode（規劃模式）

最簡單的方式。輸入 /plan 或按 Shift+Tab，Agent 會先提問、整理計畫，再開始寫程式。

2
請 Agent 先採訪你

告訴 Agent：「請先問我問題，挑戰我的假設，把模糊的想法變成清楚的需求，再開始寫程式。」

3
PLANS.md 執行計畫模板

進階用法：建立一份 PLANS.md 文件，定義多步驟任務的標準執行流程。

⚠️ 新手常犯的錯誤：跳過規劃步驟，直接叫 Agent 執行複雜任務。結果是 Agent 完成了很多工作，但方向是錯的，你要花更多時間修正。

🎯 快速測驗

當你要請 AI Agent 完成一個「複雜、模糊的任務」時，最佳做法是什麼？

📄AGENTS.md — Agent 的說明書

AGENTS.md 就像是你給 Agent 的「工作手冊」，讓它每次開始工作時都能自動讀取你的期望與規範，不需要每次重複說明。

🔑 生活比喻
想像你請了一位新工讀生。你可以每天當面交代他規則，也可以給他一本「公司員工手冊」讓他自行閱讀。AGENTS.md 就是那本手冊 — 一次寫好，Agent 每次都會遵守。

一份好的 AGENTS.md 通常包含：

  • 📁 專案結構：重要資料夾和檔案的說明
  • 🚀 如何執行專案：啟動、測試、建置的指令
  • 📏 程式規範：命名慣例、程式風格
  • 🚫 禁止事項：什麼事 Agent 不可以做
  • ✅ 完成定義：什麼叫做一個任務「完成了」
⚡ 使用 CLI 指令 /init 可以快速建立一份基礎 AGENTS.md。建立後再根據你的實際需求修改。

AGENTS.md 可以放在多個層級：

  • ~/.codex/AGENTS.md — 你的個人預設（所有專案都適用）
  • 專案根目錄 — 整個專案的規範
  • 子資料夾 — 局部的特殊規則（優先度最高）

🎯 快速測驗

關於 AGENTS.md，以下哪個說法是正確的？

🔍驗證與審查 — 不要盲目接受

Agent 產出程式碼只是開始，要求它自我驗證和審查才是完整的流程。一個好的 AI Agent 工作流程不只是「產生程式碼」，還包括：

完整的 Agent 工作迴圈

撰寫程式碼 → 撰寫 / 更新測試 → 執行測試 → 程式碼審查 → 確認完成

你可以在 AGENTS.md 或 Prompt 中指定 Agent 必須：

  • ✅ 為修改的功能撰寫或更新測試
  • ✅ 執行完整的測試套件
  • ✅ 通過 lint、型別檢查、格式化
  • ✅ 最終行為符合任務要求
  • ✅ 審查 diff，找出可能的 bug 或回歸問題
🛠️ 在 Codex App 使用 /review 指令，可以讓 Agent 審查程式碼變更 — 可以針對 PR、未提交的修改，或特定 commit 進行審查。
🏢 OpenAI 內部所有的 PR（Pull Request）都由 Codex 進行 100% 的程式碼審查！你也可以在 GitHub 整合中啟用這個功能。

Agent 完成程式碼修改後，下一步最應該做什麼？

🔗MCP — 連接外部世界

MCP（Model Context Protocol） 是一個開放標準，讓 AI Agent 能連接到外部工具和系統，例如你的資料庫、Slack、GitHub、Jira 等等。

🔑 生活比喻
MCP 就像是手機的 App Store — 你可以為你的 AI Agent 安裝「插件」，讓它能存取更多外部資訊和工具，而不只是靠你貼上的文字。

什麼時候應該使用 MCP？

  • 📦 需要的資訊存在於程式庫之外（例如 Jira ticket、資料庫）
  • 🔄 資料經常變動（例如每天的報告、即時數據）
  • 🔧 需要 Agent 呼叫工具，而不是靠貼上的指令
  • 👥 需要跨用戶、跨專案的可重複整合
⚠️ 新手建議：不要一次接上所有工具！先從 1-2 個最常用的工具開始，找到真正能省時間的整合點，再慢慢擴展。

常見的 MCP 應用場景：

  • 🐛 讀取 GitHub Issues / Jira Tickets 來修 bug
  • 📊 連接資料庫查詢即時數據
  • 💬 整合 Slack 發送通知
  • 📁 存取 Google Drive 的文件

🎯 快速測驗

你想讓 AI Agent 自動讀取最新的 Jira Bug 票，幫你修復程式碼。這種情境最適合使用什麼技術？

⚡Skills — 打包你的工作方法

當你發現自己重複使用同樣的 Prompt，或反覆糾正同樣的問題時，就是時候把它打包成一個 Skill（技能包）了。

🔑 什麼是 Skill？
Skill 是一個 SKILL.md 文件，它定義了一個可重複使用的工作方法 — 包含說明、輸入輸出格式、使用時機。Agent 讀取後就能按照這個方法一致地執行任務。

哪些工作適合做成 Skill？

  • 📋 Log 分析：每天早上分析系統錯誤日誌
  • 📝 Release Notes 撰寫：依照固定格式整理版本更新說明
  • 🔍 Code Review：按照團隊規範審查 PR
  • 📊 每週進度摘要：整理 Jira / GitHub 本週完成事項
  • 🐛 標準除錯流程：按步驟診斷常見問題類型
🚀 可以使用內建的 $skill-creator Skill 來幫你快速建立新的 Skill！第一個版本先放在本地測試，穩定後再分享給團隊。

Skill 存放位置：

  • $HOME/.agents/skills — 你的個人技能（私有）
  • 專案 .agents/skills — 團隊共用技能（可放進 Git）

🎯 快速測驗

你發現自己每週都要對 Agent 說「依照我們的格式幫我寫這週的 Release Notes」，而且每次都要重新說明格式。最好的解決方案是什麼？

⏰自動化 — 讓 Agent 在背景工作

當一個工作流程已經穩定且可預測時，就可以設定 Automation（自動化排程），讓 Agent 在背景定時執行，完全不需要你手動觸發。

🔑 Skill vs Automation 的關係
Skill 定義「怎麼做」（方法）
Automation 定義「何時做」（排程）

一個規則：Skill 先做好，再設 Automation。如果工作流程還不穩定，就不要急著自動化！

哪些工作適合設定 Automation？

  • 🌅 每天早上：摘要昨天的 commits，生成 standup 報告
  • 🔎 定期掃描：每週檢查潛在 bug 或安全漏洞
  • 📋 每次 PR：自動進行程式碼審查
  • 🚨 CI 失敗時：分析錯誤原因，提供修復建議
  • 📰 每版本：自動產生 Release Notes 草稿
💡 Automation 不只是執行任務，也適合用來「反省和維護」— 例如：每週回顧 Agent 的工作記錄，找出重複出現的問題，更新 AGENTS.md 或 Skill。
